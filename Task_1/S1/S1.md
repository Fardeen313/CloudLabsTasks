# Lab 1 – S1. Sales Trend Analysis

## Scenario

You are a data analyst at a retail company. Your manager asks:

> "Show me how much we sold each year — and how much we grew compared to the year before."

Your job is to write a SQL query that reads from the `Sales` table and gives the total sales per year along with the percentage growth from the previous year.

---

## Task

**Given table:** `Sales(SaleID, SaleDate, Amount)`

**What to produce ?:**

| SalesYear | TotalSales | PercentGrowth |
|-----------|------------|---------------|
| 2021 | 10000.00 | NULL |
| 2022 | 12000.00 | 20.00 |
| 2023 | 15000.00 | 25.00 |
| 2024 | 18000.00 | 20.00 |

NULL in the first row is correct — there is no previous year to compare against.

## Query

```sql
WITH YearlySales AS (
  SELECT
    YEAR(SaleDate) AS SalesYear,
    SUM(Amount)    AS TotalSales
  FROM Sales
  GROUP BY YEAR(SaleDate)
),
SalesWithGrowth AS (
  SELECT
    SalesYear,
    TotalSales,
    LAG(TotalSales) OVER (ORDER BY SalesYear) AS PrevYearSales
  FROM YearlySales
)
SELECT
  SalesYear,
  TotalSales,
  ROUND(((TotalSales - PrevYearSales) * 100.0 / PrevYearSales), 2) AS PercentGrowth
FROM SalesWithGrowth;
```

**How it works:**

- `YearlySales` groups sales by year and totals the amount for each year
- `LAG(TotalSales)` fetches the previous year's total so you can compare
- `ROUND(..., 2)` gives the growth as a percentage to 2 decimal places
- First year returns NULL for growth because there is nothing before it

## Validation

The validator compares your exported CSV against the saved answer file.

### What it checks

- All 4 year rows are present
- `TotalSales` matches per year
- `PercentGrowth` matches per year

### Why "close but correct" still passes

The dataset uses round numbers on purpose:

```
2022 growth = (12000 - 10000) / 10000 × 100 = 20.00   exact
2023 growth = (15000 - 12000) / 12000 × 100 = 25.00   exact
2024 growth = (18000 - 15000) / 15000 × 100 = 20.00   exact
```

Because the math always lands on a clean number, small formatting differences do not cause false failures.

### Tolerance rules

| What differs | Example | Pass? |
|---|---|---|
| Decimal places | `20.0` vs `20.00` | Yes — both rounded to 2dp before compare |
| Tiny rounding | `20.001` vs `20.00` | Yes — both rounded to 2dp before compare |
| NULL vs empty | `NULL` vs ` ` | Yes — treated the same |
| Row order | 2024 before 2021 | Yes — validator sorts both by year first |
| Column name | `PercentGrowth` vs `pct_growth` | Yes — compared by position, not name |
| NULL vs 0 | `NULL` vs `0.00` | No — means the logic is wrong |
| Wrong value | `20.00` vs `19.00` | No — incorrect calculation |
| Missing row | 3 rows instead of 4 | No — a year is missing |
| Extra rows | 5 rows instead of 4 | No — data was likely inserted twice |

### Validation logic

```python
def validate(user_csv, answer_csv):
    user   = sort_by_year(load(user_csv))
    answer = sort_by_year(load(answer_csv))

    if len(user) != len(answer):
        return FAIL, "Row count does not match"

    for i in range(len(answer)):
        u, a = user[i], answer[i]

        if u["SalesYear"] != a["SalesYear"]:
            return FAIL, f"Wrong year at row {i+1}"

        if round(float(u["TotalSales"]), 2) != round(float(a["TotalSales"]), 2):
            return FAIL, f"TotalSales mismatch at {a['SalesYear']}"

        u_growth = None if is_null(u["PercentGrowth"]) else round(float(u["PercentGrowth"]), 2)
        a_growth = None if is_null(a["PercentGrowth"]) else round(float(a["PercentGrowth"]), 2)

        if u_growth != a_growth:
            return FAIL, f"PercentGrowth mismatch at {a['SalesYear']}"

    return PASS, "Task completed successfully"
```

## Common Errors

| Error | Fix |
|---|---|
| `Invalid object name 'Sales'` | Run the CREATE TABLE and INSERT steps first |
| Growth shows `0` for all years | Change `* 100` to `* 100.0` |
| First year shows `0` instead of NULL | Use `LAG()` — do not hardcode 0 |
| Validator says row count mismatch | You inserted data twice — run `DELETE FROM Sales` and insert again |
| Wrong database | Check the database dropdown at the top of the query window |

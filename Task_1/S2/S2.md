# Lab 1 – S2. Remove Duplicates with Ranking

---

## Scenario

You are a data analyst at a company. HR gives you an employee list but it has multiple employees per department. Your manager asks:

> "From each department, give me only the highest paid employee."

Your job is to write a SQL query that ranks employees within each department by salary and returns only the top one per department.

---

## Task

**Given table:** `Employees(EmployeeID, Name, Department, Salary)`

**What to produce:**

| EmployeeID | Name | Department | Salary |
|------------|------|------------|--------|
| 3 | Emma | HR | 75000.00 |
| 5 | David | IT | 95000.00 |
| 7 | Grace | Finance | 90000.00 |
| 11 | Kevin | Marketing | 72000.00 |

One row per department. The employee with the highest salary in that department.

---

## Dataset

Run this to create and fill the table:

```sql
CREATE TABLE Employees (
    EmployeeID INT PRIMARY KEY,
    Name       VARCHAR(50),
    Department VARCHAR(50),
    Salary     DECIMAL(10, 2)
);

INSERT INTO Employees VALUES
(1,  'Alice',   'HR',        55000.00),
(2,  'Bob',     'HR',        60000.00),
(3,  'Emma',    'HR',        75000.00),
(4,  'Charlie', 'IT',        80000.00),
(5,  'David',   'IT',        95000.00),
(6,  'Frank',   'IT',        85000.00),
(7,  'Grace',   'Finance',   90000.00),
(8,  'Henry',   'Finance',   70000.00),
(9,  'Isla',    'Finance',   65000.00),
(10, 'Jack',    'Marketing', 60000.00),
(11, 'Kevin',   'Marketing', 72000.00),
(12, 'Laura',   'Marketing', 68000.00);
```

Each department has 3 employees. Only one should appear in the final output — the highest paid one.

---

## Query

```sql
WITH RankedEmployees AS (
  SELECT
    EmployeeID,
    Name,
    Department,
    Salary,
    ROW_NUMBER() OVER (
      PARTITION BY Department
      ORDER BY Salary DESC
    ) AS rnk
  FROM Employees
)
SELECT
  EmployeeID,
  Name,
  Department,
  Salary
FROM RankedEmployees
WHERE rnk = 1
ORDER BY Department;
```

**How it works:**

- `PARTITION BY Department` — restarts the ranking for each department separately
- `ORDER BY Salary DESC` — ranks employees from highest to lowest salary within each department
- `ROW_NUMBER()` — gives rank 1 to the top employee in each department
- `WHERE rnk = 1` — keeps only the top-ranked employee per department
- `ORDER BY Department` — sorts the final output alphabetically by department

## Validation

The validator compares your exported CSV against the saved answer file.

### What it checks

- All 4 department rows are present
- Correct employee selected per department
- Salary matches per department

### Why "close but correct" still passes

The dataset is designed so each department has one clear highest salary with no ties:

```
HR        →  55000, 60000, 75000   →  Emma  wins   (75000)
IT        →  80000, 85000, 95000   →  David wins   (95000)
Finance   →  65000, 70000, 90000   →  Grace wins   (90000)
Marketing →  60000, 68000, 72000   →  Kevin wins   (72000)
```

No two employees share the same salary in the same department. So `ROW_NUMBER()` always picks the same person regardless of any tie-breaking variation in the user's query.

### Tolerance rules

| What differs | Example | Pass? |
|---|---|---|
| Row order | Finance before HR | Yes — validator sorts both by Department before compare |
| Column name | `Name` vs `emp_name` | Yes — compared by position, not name |
| Salary decimal places | `75000.0` vs `75000.00` | Yes — both rounded to 2dp before compare |
| EmployeeID excluded | User selects only Name, Dept, Salary | Yes — EmployeeID is not used in comparison |
| Wrong employee selected | Bob instead of Emma for HR | No — wrong logic, lower salary selected |
| Missing department | Only 3 rows instead of 4 | No — a department is missing |
| Extra rows | More than 4 rows | No — duplicates were not removed correctly |
| Wrong salary | Grace shown with 70000 instead of 90000 | No — incorrect ranking |

### Validation logic

```python
def validate(user_csv, answer_csv):
    user   = sort_by_department(load(user_csv))
    answer = sort_by_department(load(answer_csv))

    if len(user) != len(answer):
        return FAIL, "Row count does not match"

    for i in range(len(answer)):
        u, a = user[i], answer[i]

        if u["Department"] != a["Department"]:
            return FAIL, f"Department mismatch at row {i+1}"

        if u["Name"] != a["Name"]:
            return FAIL, f"Wrong employee for {a['Department']} — expected {a['Name']}"

        if round(float(u["Salary"]), 2) != round(float(a["Salary"]), 2):
            return FAIL, f"Salary mismatch for {a['Department']}"

    return PASS, "Task completed successfully"
```

> `EmployeeID` is intentionally excluded from comparison. The user may or may not select it — what matters is the correct Name and Salary per department.

## Common Errors

| Error | Fix |
|---|---|
| `Invalid object name 'Employees'` | Run the CREATE TABLE and INSERT steps first |
| More than 4 rows in output | `WHERE rnk = 1` is missing or `ROW_NUMBER()` is not used |
| Wrong employee shown per department | `ORDER BY Salary DESC` — make sure it is DESC not ASC |
| All employees returned | The CTE filter `WHERE rnk = 1` was not applied |
| Validator says row count mismatch | Data was inserted twice — run `DELETE FROM Employees` and insert again |
